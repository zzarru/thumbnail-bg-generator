## 1. 바이브 코딩이란?

바이브 코딩은 **AI에게 자연어로 지시하여 프로그램을 만드는 방식**입니다.
코드를 직접 작성하는 대신, "이런 프로그램을 만들어줘"라고 말하면 AI가 코드를 생성합니다.

### 기존 개발 vs 바이브 코딩

| | 기존 개발 | 바이브 코딩 |
|---|---|---|
| 코드 작성 | 개발자가 직접 | AI가 작성 |
| 필요한 지식 | 프로그래밍 언어, 프레임워크 | 무엇을 만들고 싶은지 설명하는 능력 |
| 소통 방식 | 코드 | 자연어 (한국어/영어) |
| 디버깅 | 코드를 읽고 분석 | 에러 메시지를 AI에게 전달 |

### 바이브 코딩의 핵심 원칙

1. **명확하게 설명하기** - "좋은 프로그램 만들어줘" (X) / "400개 강의 썸네일을 AI로 자동 생성하는 웹 앱 만들어줘" (O)
2. **단계적으로 진행하기** - 한 번에 완성하려 하지 말고, 작은 단위로 나눠서 진행
3. **에러를 두려워하지 않기** - 에러가 나면 그대로 AI에게 보여주면 됨
4. **검증하면서 진행하기** - 중간중간 실행해보고 결과 확인

## 2. 시작하기 전에 준비할 것

### 필수 도구

| 도구 | 용도 | 설치 방법 |
|------|------|-----------|
| **Claude Code** | AI 코딩 도구 | [claude.ai/claude-code](https://claude.ai/claude-code) |
| **Python** | 프로그래밍 언어 | [python.org](https://python.org)에서 다운로드 |
| **Git** | 코드 버전 관리 | [git-scm.com](https://git-scm.com)에서 다운로드 |
| **GitHub 계정** | 코드 저장소 | [github.com](https://github.com)에서 가입 |
| **GitHub CLI** | 터미널에서 GitHub 사용 | [cli.github.com](https://cli.github.com)에서 다운로드 |

### API 키 준비

이 프로젝트에서는 외부 서비스의 API 키가 필요합니다:

- **OpenAI API 키**: [platform.openai.com](https://platform.openai.com)에서 발급
- **Google AI API 키**: [ai.google.dev](https://ai.google.dev)에서 발급
- **Supabase 프로젝트**: [supabase.com](https://supabase.com)에서 프로젝트 생성

> API 키는 비밀번호와 같습니다. 절대 다른 사람에게 공유하거나 코드에 직접 적지 마세요.

## Step 1: 프로젝트 구상하기

### 우리가 만들 것

**강의 썸네일 배경 이미지 자동 생성기**
- 400개 이상의 강의에 대해 AI로 배경 이미지를 자동 생성
- 여러 AI 모델의 결과를 비교하고 최종 선택
- 웹 브라우저에서 간편하게 사용

### Claude Code에 처음 말 걸기

Claude Code를 실행하고, 프로젝트 폴더에서 다음과 같이 대화를 시작합니다:

```
프롬프트:
"강의 썸네일 배경 이미지를 AI를 통해서 자동 생성하는 프로그램을 만들고 싶어.
이를 위한 깃헙 레포지토리를 생성하고 리드미 파일을 만들어줘"
```

> Claude가 GitHub 레포지토리를 만들고 기본 README 파일을 생성해줍니다.
> 이 시점에서 GitHub CLI 로그인이 필요할 수 있습니다 (아래 Git 매뉴얼 참고).

## Step 2: PRD(기획서) 작성하기

### PRD란?

PRD(Product Requirements Document)는 **"무엇을 만들 것인가"를 정리한 문서**입니다.
바이브 코딩에서도 이 단계가 매우 중요합니다. AI가 올바른 프로그램을 만들려면 명확한 기획이 필요합니다.

### 심층 인터뷰 방식으로 PRD 만들기

Claude Code에 다음과 같이 요청합니다:

```
프롬프트:
"PRD 작성을 위한 심층 인터뷰를 진행해줘"
```

Claude가 질문을 하나씩 던집니다. 실제로 우리가 답변한 내용을 예시로 보겠습니다:

### 인터뷰 질문과 답변 예시

**Q: 이 프로그램으로 해결하려는 핵심 문제가 뭔가요?**
```
답변:
"400여 개가 넘는 강의 썸네일 이미지를 대량으로 생산해야 하는데,
현재 팀에 디자이너가 부재한 상황이야.
그리고 썸네일 배경 이미지로 강의를 구별할 수 있도록 해야해
(이미지에 텍스트나 강사 이미지는 들어가지 않음)"
```

**Q: 이 프로그램은 누가 사용하나요?**
```
답변:
"일단은 내가 먼저 사용하고 나중에 팀원들에게 공유해서 사용할 수 있게끔 하고 싶어.
우선은 현재 레포지토리에서 개인이 사용하면서 테스트를 해보고
이후에 안정화 되면 배포되어 있는 웹에 통합하려고 해"
```

**Q: AI 이미지 생성 API는 어떤 걸 사용하고 싶으세요?**
```
답변:
"Google Imagen 또는 NanoBanana를 이용하고 싶어"
```

**Q: 실행 방식은 어떤 형태를 원하세요?**
```
답변: "B" (간단한 웹 UI)
```

### 핵심 포인트

- 한 번에 모든 것을 설명하지 않아도 됩니다
- Claude가 질문하면 하나씩 답변하면 됩니다
- 모르는 부분은 "추천해줘"라고 해도 됩니다
- 기술적 결정은 Claude가 제안하고, 여러분이 선택하면 됩니다

### PRD 결과물

인터뷰가 끝나면 Claude가 PRD 문서를 자동으로 작성합니다.
이 프로젝트에서는 다음 내용이 포함되었습니다:

- 문제 정의 (왜 이 프로그램이 필요한가)
- 사용자 시나리오 (어떻게 사용하는가)
- 기술 스택 (어떤 기술을 쓰는가)
- 데이터 소스 (어디서 데이터를 가져오는가)
- AI 모델 구성 (어떤 AI를 쓰는가)
- UI 구성 (화면이 어떻게 생겼는가)
- 프로젝트 구조 (파일이 어떻게 구성되는가)
- 개발 단계 (Phase 1, 2, 3)

## Step 3: 구현 계획 세우기

PRD가 완성되면 Claude가 자동으로 구현 계획을 제안합니다.

```
프롬프트:
"오케이 계속 진행해보자"
```

Claude가 총 8개의 Task로 구현 계획을 세웁니다:

| Task | 내용 | 설명 |
|------|------|------|
| 1 | 프로젝트 초기 설정 | 필요한 패키지 목록, 설정 파일 |
| 2 | YAML 템플릿 파일 | AI에게 보낼 프롬프트 템플릿 |
| 3 | DB Reader | 데이터베이스에서 강의 정보 가져오기 |
| 4 | Prompt Builder | 템플릿 + 강의 정보 = 최종 프롬프트 |
| 5 | Image Generator | AI API 호출하여 이미지 생성 |
| 6 | Logger | 생성 이력 기록 |
| 7 | Streamlit UI | 웹 화면 만들기 |
| 8 | README | 사용 설명서 |

### 실행 방식 선택

```
프롬프트:
"Subagent-Driven로 진행하자"
```

> Subagent-Driven 방식은 각 Task를 독립적으로 실행하는 방식입니다.
> 빠르고 효율적이지만, "Inline Execution"(순차 실행)도 가능합니다.

## Step 4: 코드 구현하기

이 단계에서는 **여러분이 할 일이 거의 없습니다.**
Claude가 계획대로 코드를 작성하고, 테스트하고, 커밋합니다.

### 실제로 일어나는 일

1. Claude가 테스트 코드를 먼저 작성 (TDD 방식)
2. 테스트가 실패하는지 확인
3. 실제 코드를 작성
4. 테스트가 통과하는지 확인
5. Git에 커밋

### 여러분이 해야 할 일

- Claude가 작업하는 동안 **기다리기**
- 중간에 질문이 오면 **답변하기**
- 완료되면 **결과 확인하기**

### 진행 상황 확인

Claude가 각 Task를 완료할 때마다 상태를 업데이트합니다:

```
Task 1: 프로젝트 초기 설정 ✅
Task 2: 설정 및 YAML 템플릿 ✅
Task 3: DB Reader ✅
Task 4: Prompt Builder ✅
Task 5: Image Generator ✅
Task 6: Logger ✅
Task 7: Streamlit UI ✅
Task 8: README ✅
테스트: 14/14 통과
```

## Step 5: 테스트 및 수정하기

### 환경 변수 설정

프로그램을 실행하기 전에 API 키를 설정해야 합니다.

1. 프로젝트 폴더에서 `.env.example` 파일을 복사하여 `.env` 파일을 만듭니다
2. `.env` 파일을 열고 API 키를 입력합니다:

```
GOOGLE_API_KEY=여기에_구글_API_키_입력
OPENAI_API_KEY=여기에_OpenAI_API_키_입력
SUPABASE_URL=여기에_Supabase_URL_입력
SUPABASE_ANON_KEY=여기에_Supabase_키_입력
```

### 프로그램 실행

```bash
streamlit run app.py
```

브라우저에서 `http://localhost:8501`로 접속합니다.

### 에러가 발생했을 때

바이브 코딩의 핵심은 **에러를 그대로 Claude에게 전달하는 것**입니다.

실제로 이 프로젝트에서 발생한 에러들:

**에러 1: API 할당량 초과**
```
프롬프트:
"Nano Banana (Efficient): 429 RESOURCE_EXHAUSTED.
{'error': {'code': 429, 'message': 'You exceeded your current quota...'}}"
```
> Claude가 해당 모델이 무료 티어에서 사용 불가능한 것을 알려주고 대안을 제시했습니다.

**에러 2: 지원하지 않는 이미지 비율**
```
프롬프트:
"Imagen 4.0: 400 INVALID_ARGUMENT.
aspectRatio 3:2 is not supported. Supported values are 1:1, 9:16, 16:9, 4:3, 3:4.
이미지 비율은 그냥 16:9로 해줘"
```
> Claude가 즉시 코드를 수정했습니다.

**에러 3: OpenAI API 파라미터 오류**
```
프롬프트:
"GPT Image (gpt-image-1): Error code: 400 - Unknown parameter: 'response_format'.
DALL-E 3: Error code: 400 - Invalid value: '1536x1024'.
이거 확인해줘. 각 모델에 맞는 파라미터랑 값을 확인했어?"
```
> Claude가 공식 문서를 확인하고 모델별로 다른 파라미터를 적용했습니다.

### 에러 대처 팁

1. **에러 메시지를 그대로 복사해서 붙여넣기** - 해석하려고 하지 마세요
2. **스크린샷도 유용** - Claude Code는 이미지도 이해합니다
3. **"이거 확인해줘"** - 간단한 한마디로 충분합니다
4. **공식 문서 URL 제공** - "문서는 https://... 여기야"라고 알려주면 Claude가 직접 확인합니다

### 새로운 기능 추가하기

기존 AI 모델에 문제가 있어서 OpenAI 모델을 추가한 예시:

```
프롬프트:
"일단 기능이 정상 작동하는지 알고 싶은데
우선 openai api를 사용해볼까...? 모델 추가하는 거 어때?"
```

> Claude가 자동으로:
> 1. OpenAI SDK 설치
> 2. 새로운 모델 설정 추가
> 3. 이미지 생성 코드에 OpenAI 지원 추가
> 4. 환경 변수 추가
> 5. 문서 업데이트

## Git 매뉴얼 (Claude Code용)

### Git이란?

Git은 **코드의 변경 이력을 관리하는 도구**입니다.
게임의 "세이브 포인트"와 비슷합니다 - 언제든 이전 상태로 돌아갈 수 있습니다.

### 기본 용어

| 용어 | 의미 | 게임 비유 |
|------|------|-----------|
| **Repository (레포)** | 프로젝트 저장소 | 게임 폴더 |
| **Commit (커밋)** | 변경사항 저장 | 세이브 |
| **Push (푸시)** | GitHub에 업로드 | 클라우드 세이브 |
| **Pull (풀)** | GitHub에서 다운로드 | 클라우드에서 불러오기 |
| **Branch (브랜치)** | 독립적인 작업 공간 | 별도 세이브 슬롯 |
| **Issue (이슈)** | 할 일 / 버그 목록 | 퀘스트 목록 |

### Claude Code에서 Git 사용하기

#### 1. GitHub 로그인

처음 한 번만 하면 됩니다. 터미널에서 직접 실행:

```bash
gh auth login
```

실행 후 나오는 옵션:
1. **GitHub.com** 선택
2. **HTTPS** 선택 (방화벽 환경에서도 잘 됨)
3. **Login with a web browser** 선택
4. 브라우저에서 인증 완료

#### 2. 레포지토리 만들기

```
Claude Code 프롬프트:
"깃헙 레포지토리를 생성해줘. 이름은 thumbnail-bg-generator로 해줘"
```

#### 3. 코드 저장하기 (커밋)

```
Claude Code 프롬프트:
"지금까지 변경한 내용을 커밋해줘"
```

> Claude가 변경 내용을 분석하고 적절한 커밋 메시지를 자동으로 작성합니다.

#### 4. GitHub에 올리기 (푸시)

```
Claude Code 프롬프트:
"커밋 푸시해줘"
```

#### 5. 커밋 + 푸시 한번에

```
Claude Code 프롬프트:
"커밋 푸시도 해줘"
```

#### 6. GitHub 이슈 만들기

```
Claude Code 프롬프트:
"이 작업 내용을 깃헙 이슈에 등록해줘.
작업 배경, 작업 내용, 인수 조건, 의존성이 포함되어야 해"
```

#### 7. 이슈 확인하기

```
Claude Code 프롬프트:
"현재 깃헙 이슈 목록 보여줘"
```

#### 8. 변경 이력 확인하기

```
Claude Code 프롬프트:
"최근 커밋 이력 보여줘"
```

#### 9. 현재 상태 확인하기

```
Claude Code 프롬프트:

```

### Git 관련 주의사항

1. **`.env` 파일은 절대 커밋하지 마세요** - API 키가 들어있습니다. `.gitignore`에 이미 등록되어 있습니다.
2. **커밋은 자주 하세요** - 작업 단위가 끝날 때마다 커밋하면 안전합니다.
3. **푸시 전에 확인하세요** - "어떤 파일이 변경되었는지 보여줘"라고 물어보세요.
